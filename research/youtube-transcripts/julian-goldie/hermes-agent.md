Brand new update from Hermes today: Hermes Agent V0.18 (The Judgment Release)\[cite: 1]. This update covers exactly what it means, how it works, and examples of how it can be used in practice\[cite: 1]. The first big highlight is the Mixture of Agents system\[cite: 1].



\## Segmen 2: Mixture Of Agents

Mixture of Agents is described more like a system rather than a single model\[cite: 1]. Inside Agent OS, Mixture of Agents functions as a way of getting better quality outputs by combining multiple models together, for example: Claude Opus 4.8 and GPT 5.5 working together\[cite: 1]. It fuses the answers of two different models to create a better output, based on the idea that "two minds are better than one"\[cite: 1].



\## Segmen 3: Benchmarks And Tradeoffs

Based on "Goldie Bench" side-by-side performance tests, Hermes Mixture of Agents ranks right at the top, even outperforming Fable 5 in many ways\[cite: 1]. This setup is basically "one-shot" because it doesn't go back and forth in a chat like Claude CLI\[cite: 1]. It takes about 10 to 15 minutes to generate a decent output directly into the workspace\[cite: 1]. It serves as an alternative to Fusion from Open Router\[cite: 1].



\## Segmen 4: Goal Mode Looping

An improved system where the agent verifies its own work via the /go or Goal Mode system\[cite: 1]. Users don't just prompt the agents; they give the Hermes agent a task, and then it loops\[cite: 1]. A separate Judge API judges the quality of the outputs generated by the Builder API\[cite: 1]. It iterates autonomously round and round based on the defined "definition of done" without requiring manual quality control checking\[cite: 1].



\## Segmen 5: Why Use Agent OS

Using Hermes inside a standard terminal makes it difficult to see previous creations and segment tasks\[cite: 1]. An Agent Operating System (Agent OS) is preferred to chat through different profiles, such as using a Hermes Oracle for news, or utilizing specific mini-apps for lead generation and outreach\[cite: 1].



\## Segmen 6: Learn And Journey

\* \*\*`/learn` command:\*\* Allows the agent to ingest a guide or a GitHub repository link, break it down, and save it as a reusable skill for future sessions\[cite: 1].

\* \*\*`/journey` command:\*\* Generates a playable timeline and breakdown of everything Hermes has learned\[cite: 1].



\## Segmen 7: Updating And Fan Out

There are three ways to update Hermes to the latest version\[cite: 1]:

1\. Asking directly inside the chat to update\[cite: 1].

2\. Clicking "Update Hermes" on the dashboard\[cite: 1].

3\. Running "Hermes updates" in the terminal\[cite: 1].



\* \*\*Background Fan Out:\*\* Allows users to delegate a task so sub-agents run asynchronously in the background without blocking the main chat\[cite: 1].

\* \*\*Obsidian Integration:\*\* Used as a free external app to visualize a galaxy map of memories and recent conversations\[cite: 1].



\## Segmen 8: Delegating Sub Agents

The system automatically picks up tasks as a skill to delegate to a team of sub-agents working in parallel\[cite: 1]. For example, in an SEO funnel use case, you can provide one keyword, and it will automatically work on five articles in parallel and deploy them\[cite: 1].

The standing gold loop keeps judging work against upfront completion evidence\[cite: 1].
